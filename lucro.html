Etapa 01
Organizar e otimizar o projeto
Prepare os arquivos antes de qualquer coisa
▾

Antes de mostrar qualquer coisa ao cliente, o projeto precisa estar limpo, organizado e funcionando sem dependências locais. Um arquivo HTML bagunçado vai gerar retrabalho e passar uma imagem ruim.

Ferramentas recomendadas
🛠 VS Code ✅ Prettier 🔍 HTML Validator (W3C) ⚡ Squoosh (imagens) 📁 FileZilla
Checklist de preparação
  • ✅ Todos os links e imagens usam caminhos relativos (sem C:\Users\...)
  • ✅ CSS e JS estão em arquivos separados ou inline — sem CDNs que dependem de internet externa
  • ✅ Imagens comprimidas (use Squoosh ou TinyPNG)
  • ✅ Validado no validator.w3.org
  • ✅ Responsivo: testado em 375px, 768px e 1280px
  • ✅ Favicon incluído
  • ✅ Meta tags (title, description, og:image) preenchidas
💡 Dica profissional: Sempre teste abrindo o arquivo HTML diretamente no navegador sem servidor local. Se quebrar, tem dependência escondida.

Etapa 03
Registrar o domínio
O endereço digital do cliente na internet
▾

O domínio é o www.nomedocliente.com.br — sem isso, o site é só um link genérico e sem credibilidade. Você pode registrar pelo cliente ou oferecer como serviço incluso.

Onde registrar
🇧🇷 Registro.br (domínios .com.br — R$40/ano) 🌎 GoDaddy (.com — ~R$50/ano) ⚡ Cloudflare (mais barato no .com) 🏠 Hostinger (combo domínio+hospedagem)
Como apontar o domínio para a hospedagem
  • 1️⃣ Registre o domínio no provedor escolhido
  • 2️⃣ No painel da hospedagem, copie os nameservers (ex: ns1.hostinger.com)
  • 3️⃣ No painel do domínio, substitua os nameservers pelos da hospedagem
  • 4️⃣ Aguarde a propagação: de 5 minutos até 48 horas
  • 5️⃣ Ative o SSL (HTTPS) — gratuito via Let's Encrypt na maioria das hospedagens
⚠️ Atenção: Sempre registre o domínio no nome do cliente, não no seu. Se o relacionamento terminar, o domínio pertence a quem pagou — e deve ser ele.

Etapa 04
Fazer o deploy (publicar)
Colocar o projeto no ar de forma profissional
▾

Existem duas formas principais de publicar um projeto HTML puro: via arrasta e solta (Netlify/Vercel) ou via FTP (hospedagem tradicional). Escolha conforme a hospedagem decidida na etapa 2.

Método 1 — Netlify Drop (mais fácil)
  • 🌐 Acesse app.netlify.com/drop
  • 📂 Arraste a pasta do projeto para a área indicada
  • ✅ Pronto! Em segundos você tem um link funcional com HTTPS
  • 🔗 Configure o domínio personalizado em Site Settings → Domain Management
Método 2 — FTP com FileZilla
  • 1️⃣ Baixe e instale o FileZilla (gratuito)
  • 2️⃣ No painel da hospedagem, crie um usuário FTP e anote: host, usuário, senha e porta
  • 3️⃣ No FileZilla: Arquivo → Gerenciador de Sites → Novo Site
  • 4️⃣ Cole os dados e clique em Conectar
  • 5️⃣ Navegue até a pasta public_html e arraste seus arquivos para lá
  • 6️⃣ Aguarde o upload e acesse o domínio no navegador
⚡
Netlify Drop
5 minutos. Zero configuração. Perfeito para iniciantes.
🔧
FTP
Mais controle. Necessário em hospedagens tradicionais (cPanel).
🤖
GitHub + CI
Deploy automático a cada commit. Ideal para projetos em andamento.

Etapa 05
Entregar ao cliente
Apresentação, acesso e documentação
▾

A entrega é onde muitos freelancers perdem pontos. Entregar bem é tão importante quanto desenvolver bem — é o que o cliente vê e lembra.

O que entregar ao cliente
🔗
Link do site
O endereço final com domínio personalizado e HTTPS ativo.
📦
Arquivos fonte
Pasta comprimida (.zip) com todo o código. Direito do cliente.
🔑
Acessos
Login FTP, painel da hospedagem e registrador de domínio.
📄
Mini manual
PDF simples explicando o que pode e o que não pode editar.
Script de entrega (mensagem modelo)
📩 Mensagem para enviar ao cliente: "Olá [Nome]! Seu site já está no ar: www.seusite.com.br 🎉
Em anexo envio a pasta com todos os arquivos e um guia rápido de uso.
Os dados de acesso à hospedagem estão no documento em PDF.
Qualquer ajuste nos primeiros 15 dias está incluso. Depois disso, seguimos com o plano de manutenção combinado."
  • 📸 Faça um vídeo de 2–3 min navegando pelo site pronto (Loom é grátis)
  • 📋 Documente tudo por escrito (e-mail ou PDF) — protege ambos os lados
  • ✍️ Assine o termo de aceite / conclusão do projeto
  • ⭐ Peça um depoimento ou avaliação Google enquanto está em alta

Etapa 06
Manutenção e receita recorrente
Transforme cada projeto em renda mensal
▾

O verdadeiro negócio não é no projeto único — é na mensalidade. Um cliente de site pode gerar renda por anos com um plano de manutenção bem estruturado.

Modelos de plano de manutenção
🥉 Básico
R$79
Hospedagem + backup mensal + suporte por e-mail
🥈 Profissional
R$149
+ 2h de alterações + relatório de visitas mensal
🥇 Premium
R$299
+ Prioridade + SEO básico + 5h de alterações/mês
O que inclui a manutenção de HTML
  • 🔒 Renovação do domínio e manutenção do SSL ativo
  • 💾 Backup dos arquivos (mensalmente no Google Drive ou similar)
  • ✏️ Pequenas alterações de texto, imagens ou cores
  • 📊 Relatório do Google Analytics (visitas, origem do tráfego)
  • ⚡ Monitoramento de velocidade e disponibilidade (UptimeRobot — grátis)
  • 🛡️ Verificação de links quebrados mensalmente
🚀 Meta realista: 15 clientes × R$149/mês = R$2.235 recorrentes com ~5–8h de trabalho por mês. Você pode automatizar os backups e relatórios com ferramentas gratuitas.
Ferramentas para automatizar
📈 Google Analytics 🔔 UptimeRobot ☁️ Google Drive (backup) 🧾 Stripe / Hotmart (cobrança recorrente) 🤖 Zapier (automações)
